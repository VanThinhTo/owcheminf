Learners as Scorers
===================

In Orange, compatible learners can be connected to feature-ranking workflows
as scorers. Molecular descriptors and fingerprint bits can therefore be ranked
with generic Orange modeling tools after they are generated by OWChemInf.

For QSAR work, feature scoring must be part of the validation process. Ranking
features once on the full dataset and then cross-validating a model produces
optimistically biased estimates.

Recommended pattern
-------------------

.. code-block:: text

   Mol Descriptors 2
       → QSAR Descriptor Explorer
       → training-only feature scoring/selection
       → learner
       → validation

Use :doc:`../widgets/model-explanation` after fitting when the goal is to
describe model behavior. Feature importance is not evidence of causality.
